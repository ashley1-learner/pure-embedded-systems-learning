History and Standardization of 'C'
  The C programming language was developed in AT&T Labs by Brian W.Kernighan and Dennis Ritchie and it was known as K&RC.
  During the year 1970, 'C' programming became very popular but without any serious standardization to the language.
  The non-official standard was called K&RC, which led to many ambiguities among different compiler programmers and led to non-portable codes.
  K&RC was the first non-official C standard.
  In 1989, American National Standard Institute (ANSI) designed and approved first C standard called X3.159-1989 and in 1990 it was approved by ISO as an international standard for C programming language.: ISO/IEC 9899:1990 (This is also called ANSI C/ C89/ C90 standard in short)
  C11 is an informal name for ISO/IEC 9899:2011 which is a new standard approved in December 2011. C11 supersedes the C99 standard. 
  In this course, we will be using the 'C11 standard' with some compilers (GCC) extensions (gnu11). More on this, we will see later. 
  Note that if you have written a code in C90 standards, then it will compile without any issue in the -std=C99 compilation (backward compatibility). If you have written a code using C99 standard specific features, then it may not compile successfully in the C90 compilation.
